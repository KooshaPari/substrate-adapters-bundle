## 1. What this repo is

`pheno-context` is the **canonical request context** substrate for the pheno-* fleet. It defines the `Context` struct, the `ContextBuilder` fluent API, and the `Context::from_headers` extractor that maps `X-Request-ID` / `X-Trace-ID` / `X-Span-ID` / `X-User-ID` / `X-Org-ID` into a typed, `Clone` + `Debug` + `PartialEq` value carried through every service that handles HTTP requests. It is a pure, single-crate Rust library — no async runtime, no I/O, no FFI — designed to be embedded in any `pheno-*-lib` consumer (notably `pheno-otel`, `pheno-tracing`, `pheno-config`).

**Why it exists:** the fleet's cross-service trace correlation (request_id → trace_id → span_id → user_id → org_id → metadata bag) had been re-implemented per service. ADR-013, ADR-015, and ADR-035 converge on a single canonical `Context` substrate so that trace propagation, OTLP export, and audit-log correlation are all reading from the same shape.

**Primary consumer:** any `pheno-*-lib` or `phenotype-*-sdk` that handles a request and needs to attach a typed context to a span, a log line, or a downstream call. The substrate MUST NOT add an async runtime, MUST NOT call into a service, and MUST NOT depend on anything heavier than `thiserror` + `http`.

## 2. Public API (canonical)

```rust
pheno_context::Context {
    request_id: String,   // required
    span_id: String,      // required
    trace_id: String,     // required
    user_id: Option<String>,
    org_id: Option<String>,
    metadata: HashMap<String, String>,  // extensible bag
}

pheno_context::ContextBuilder::new() -> ContextBuilder
    .with_request_id(impl Into<String>) -> Self
    .with_span_id(impl Into<String>)    -> Self
    .with_trace_id(impl Into<String>)   -> Self
    .with_user_id(impl Into<String>)    -> Self
    .with_org_id(impl Into<String>)     -> Self
    .with_metadata(k: impl Into<String>, v: impl Into<String>) -> Self
    .build() -> Result<Context, ContextError>

pheno_context::Context::new() -> ContextBuilder
pheno_context::Context::from_headers(&HeaderMap) -> Result<Context, ContextError>
pheno_context::ContextError::MissingHeader(String)
```

Full API surface: see `SPEC.md` § 4 and `llms.txt`. Tests: 5 unit tests in `crates/pheno-context/src/lib.rs:181-274`.

- **No `unwrap()` in lib code** (ADR-023 Rule 3.1). Use `?` + `ContextError`. Allowed in `#[cfg(test)] mod tests` only.
